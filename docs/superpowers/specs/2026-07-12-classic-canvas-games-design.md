# 经典 Canvas 游戏扩展设计

## 目标

在 HooksVue 的“作品展示”中新增四款可直接在弹窗中游玩的经典小游戏：90 坦克风格、打砖块、飞翔小鸟和太空战机。保持现有 Vue 3 单文件组件与 Canvas 2D 技术路线，不增加第三方运行时依赖或外部图片素材。

## 架构

每款游戏使用一个独立组件，内部拥有自己的状态、动画循环、输入处理、碰撞检测和绘制逻辑：

- `src/components/games/BattleCityGame.vue`
- `src/components/games/BrickBreakerGame.vue`
- `src/components/games/FlappyBirdGame.vue`
- `src/components/games/SpaceShooterGame.vue`

`src/App.vue` 负责导入组件，并在现有 `openGame(work)` 分支中通过唯一 `type` 选择游戏。`src/utlis/menuItems.js` 的 `onlineWorksList` 负责提供名称、简介、图标和对应 `type`。各游戏不共享可变状态，不改动现有游戏组件。

## 游戏规则

### 90 坦克风格

- 400×400 顶视角战场，玩家守护底部基地。
- 玩家使用方向键或 WASD 四向移动，空格键射击。
- 地图包含不可穿越边界、可被子弹摧毁的砖墙和基地防线。
- 敌军按波次生成，自动移动、避障并间歇射击。
- 玩家子弹消灭敌军并计分；敌军子弹命中玩家或基地时失败。
- 消灭本关规定敌军数量后获胜，可重新开始。

### 打砖块

- 玩家移动底部挡板反弹小球，清除上方砖块。
- 球落出底边扣除生命；生命耗尽失败。
- 清空砖块后进入胜利状态；分数和球速随击破数量增长。

### 飞翔小鸟

- 空格键、点击画布或触控按钮使小鸟向上跃起，重力持续下拉。
- 管道从右向左移动并随机生成缺口。
- 穿过一组管道得分；碰到管道、地面或顶部时结束。

### 太空战机

- 玩家在画面底部区域移动战机并持续或主动射击。
- 敌机按波次从顶部进入，具有不同移动速度和生命值。
- 子弹命中敌机得分；敌机或敌方子弹命中玩家扣除生命。
- 生命耗尽后结束，随分数提高生成频率和难度。

## 输入与界面

- 桌面端支持方向键/WASD；射击或动作键使用空格。
- 每款游戏提供屏幕方向与动作按钮，供触屏设备操作。
- 弹窗内显示标题、分数、生命或目标进度、暂停/开始状态、重新开始按钮和简短操作说明。
- Canvas 通过 CSS 限制最大宽度并按比例缩放，逻辑坐标保持固定，避免移动端改变碰撞计算。
- 游戏视觉使用 Canvas 基础图形、像素色块和少量 emoji/文本，不加载远程资源。

## 生命周期与持久化

- 使用 `requestAnimationFrame` 驱动实时游戏；定时生成行为使用逻辑时间戳，避免堆叠多个 interval。
- 组件挂载时注册键盘事件，卸载时移除监听并取消动画帧。
- 暂停、失败或胜利时停止状态更新，但保留当前画面。
- 每款游戏使用独立 `localStorage` 键保存最高分；读取异常或无效值时回退为 0。

## 异常与边界

- 防止按键默认滚动页面，并忽略游戏结束后的移动/射击输入。
- 新一局必须完整重置实体、计时器、分数、生命、关卡和动画时间。
- 子弹、敌机、管道等离开画面后及时移除，避免数组无限增长。
- 碰撞处理一次只结算一次得分或伤害，防止同一帧重复命中。

## 验证

- 数据测试确认四个 `onlineWorksList` 条目及 `type` 唯一且存在。
- 静态测试确认 `App.vue` 已导入四个组件并包含对应 `openGame` 分支。
- 对可抽离的纯逻辑（矩形碰撞、圆形碰撞或关键状态转换）使用 Node 测试覆盖。
- 运行现有导航/MCP 测试和生产构建。
- 浏览器烟雾测试依次打开四款游戏，检查开始、键盘输入、触屏按钮、暂停、重新开始及弹窗关闭后的事件清理。
