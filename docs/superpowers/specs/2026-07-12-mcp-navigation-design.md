# MCP 导航分类设计

## 目标

在 HooksVue 导航站中新增两个一级分类，为用户提供数量充足、可直接访问且用途清晰的 MCP 相关资源：

- `MCP Skills`：Agent Skills 官方仓库、厂商技能库、社区技能集合及技能发现工具。
- `MCP Servers`：可供 MCP 客户端连接或本地安装的服务端项目、官方 Registry 与优质资源目录。

## 数据结构

沿用 `src/utlis/menuItems.js` 中 `menuItemsList` 的现有结构，在当前分类后追加：

- ID `22`，名称 `MCP Skills`，图标 `🧠`。
- ID `23`，名称 `MCP Servers`，图标 `🔌`。

每个资源使用现有卡片兼容字段：`id`、`name`、`desc`、`icon`、`link`、`platform`、`price`。ID 使用 `mcp-skill-*` 和 `mcp-server-*` 前缀，避免与已有项目冲突。

## 收录标准

- 优先收录官方组织、活跃厂商和成熟社区维护的 GitHub 仓库或官网。
- Skills 覆盖官方示例、云平台、AI/数据、开发工具、文档办公及社区集合。
- Servers 覆盖文件与代码、数据库、浏览器与搜索、云平台、协作办公、自动化和资源聚合。
- 对内容重复、来源不明、明显停止维护或链接失效的项目不重复收录。
- 描述使用简短中文，说明核心用途；平台与价格只填写能够合理确认的信息。

## 交互与视觉

不新增组件、不改动布局和样式。新分类自动使用现有侧边栏、搜索、卡片、收藏、右键复制链接和新标签页打开能力。这样可保持桌面端与移动端行为一致，并避免影响用户已有修改。

## 异常与兼容

- 外部项目日后失效时，页面仍正常渲染，只会由目标站点返回错误。
- 对需要 API Key、账号或本地安装的资源，在描述中明确提示其性质，不在导航页保存任何密钥。
- 数据只包含普通 HTTPS 链接，不引入运行时依赖或远程脚本。

## 验证

- 运行生产构建，确保 Vue 数据文件语法正确且页面可打包。
- 检查新增分类 ID 和工具 ID 唯一。
- 检查所有新增链接均为 HTTPS，并对主要来源执行联网可访问性抽查。
- 确认原有分类与用户当前未提交修改未被覆盖。
