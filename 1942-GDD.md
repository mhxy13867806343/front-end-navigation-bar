# 1942 复刻版 · 游戏设计文档 (GDD)

> 版本：v0.2 (2026-07-23) — 当前版
> 作者：GameDesigner
> 目标平台：浏览器 (HTML5 Canvas，桌面键盘 + 移动触屏)
> 原型来源：Capcom《1942》(1984) 纵版卷轴射击 (STG)
> 技术实现：纯原生 HTML/JS/Canvas，单文件，无引擎、无第三方依赖

---

## 1. 设计支柱 (Design Pillars)

所有后续设计决策都对照这 4 条支柱做取舍，任何新功能若违反支柱则需重写或砍掉。

| # | 支柱 | 非妥协的玩家体验 | 取舍准则 |
|---|------|------------------|----------|
| P1 | **上手即紧张** | 30 秒内看懂"移动+射击+躲"，但立刻感受到弹幕压力 | 不加入需要说明书的机制；新机制必须有即时正反馈 |
| P2 | **风险回报式成长** | 拿到武器升级感到"变强了"，死掉掉级感到"心疼" | 强化稀缺、掉级有痛感；不做无脑滚雪球 |
| P3 | **空中桶滚 (Loop)** | 高手靠操作而非运气脱险，有一个"救命"炫技闪避 | Loop 必须有充能限制 + 后摇，不能当无敌外挂 |
| P4 | **街机式刷分** | 连杀有倍率，分数驱动反复挑战 | 连击是核心反馈，掉连击的惩罚要清晰可见 |

## 2. 核心循环 (Core Gameplay Loop)

```markdown
# Core Loop: 1942 Remake

## Moment-to-Moment (0–30 秒)
- Action: 方向键/WASD 移动飞机，按住射击键自动开火
- Feedback: 命中→爆炸粒子 + 短促音效；击杀飘字得分
- Reward: 即时击杀得分 + 连击倍率上涨（视觉/听觉正反馈）

## Session Loop (1–3 分钟 / 一关)
- Goal: 刷怪阶段存活清场，击败关底 Boss
- Tension: 弹幕密度 + 敌人冲撞 + Loop 充能见底时的决策压力
- Resolution: 击败 Boss→大额得分 + 武器保级 + Loop 回充；死亡则掉命并掉一级武器

## Long-Term Loop (多局 / 多关)
- Progression: 关卡难度曲线递增（刷怪频率↑、敌人速度↑、Boss 血量↑），无限关兜底
- Retention Hook: 本地最高分榜 + 每 5 关精英 Boss 里程碑
```

## 3. 机制规格 (Mechanic Specifications)

### 3.1 移动 (Movement)
- **Purpose**: 基础空间控制，构成所有躲弹决策的前提
- **Player Fantasy**: 灵活穿梭于弹幕之间的老练飞行员
- **Input**: 方向键 / WASD（八向）；触屏为拖拽飞机
- **Output**: 玩家坐标按速度积分更新，钳制在屏幕内
- **Success Condition**: 输入与飞机位移 1:1 跟手，无延迟感
- **Failure State**: 输入死区、边界穿模、斜向速度 > 直线速度（需向量归一化）
- **Edge Cases**: 同时按相反方向→抵消；触屏松手→飞机停在当前位置
- **Tuning Levers**: `PLAYER_SPEED`
- **Dependencies**: 碰撞系统、Loop 动画

### 3.2 主武器射击 (Primary Fire)
- **Purpose**: 提供持续输出，是清场与得分的主通道
- **Player Fantasy**: 火力压制的安全感
- **Input**: 按住 J / 空格（桌面）；触屏自动开火
- **Output**: 按 `FIRE_INTERVAL` 周期从机头发射子弹（数量/角度随武器等级变化）
- **Success Condition**: 开火节奏稳定，不卡顿、不漏帧
- **Failure State**: 蓄力误解、弹药穿模、等级切换时火力断档
- **Edge Cases**: 武器满级再吃 POW→不溢出，给少量加分补偿
- **Tuning Levers**: `FIRE_INTERVAL`、`WEAPON_TIERS[]`
- **Dependencies**: 敌人血量、Boss 血量（反推需要多少 DPS）

### 3.3 空中桶滚 (Loop / 招牌闪避)
- **Purpose**: 给高手一个"操作脱险"的出口，承载 P3 支柱
- **Player Fantasy**: 一个炫酷 barrel-roll 躲掉致命弹幕
- **Input**: K / Shift（桌面）；触屏右下角按钮
- **Output**: 触发 1.3s 无敌 + 桶滚旋转动画；消耗 1 充能；有 0.4s 后摇
- **Success Condition**: 玩家在弹幕中靠 Loop 存活，且有"我秀了"的爽感
- **Failure State**: 充能无限→变成无敌外挂；或后摇过长→Loop 反而送命
- **Edge Cases**: 充能为 0 时按键→无效并给"无充能"反馈音；Loop 中再按→忽略
- **Tuning Levers**: `LOOP_DURATION`、`LOOP_COOLDOWN`、`LOOP_START`(星)
- **Dependencies**: 无敌状态、敌人子弹碰撞判定

### 3.4 武器升级 (POW Power-up)
- **Purpose**: 兑现 P2 的"变强"承诺，制造稀缺与成长
- **Player Fantasy**: 捡到 POW 后火力质变
- **Input**: 碰触掉落的 POW 道具
- **Output**: `weaponTier` +1（上限 3）；满级时转为 +500 分
- **Success Condition**: 玩家明确感知到火力变化（弹道数量/角度变化）
- **Failure State**: 掉率过高→无意义；掉率过低→全程刮痧
- **Edge Cases**: 死亡掉级（−1 级，最低 0）；关卡通关保级
- **Tuning Levers**: `POW_DROP_CHANCE`、`WEAPON_TIERS[]`
- **Dependencies**: 主武器射击、得分系统

### 3.5 生命与受伤 (Lives & Damage)
- **Purpose**: 提供失败成本与重试节奏
- **Player Fantasy**: "再来亿次"的街机韧性
- **Input**: 被敌弹/敌机撞击
- **Output**: 掉 1 命 + 掉 1 级武器 + 2.0s 无敌闪烁；命数 0→Game Over
- **Success Condition**: 受伤反馈清晰（闪红、震屏、掉级提示），且不会"冤死"
- **Failure State**: 复活即被弹幕包围→连环死；或受伤无惩罚→无风险
- **Edge Cases**: 受伤瞬间清除附近敌弹，避免复活即死；命数上限 5
- **Tuning Levers**: `START_LIVES`、`HIT_INVULN`、`LIVES_CAP`
- **Dependencies**: 武器系统、Game Over 状态

### 3.6 连击系统 (Combo / Score Multiplier)
- **Purpose**: 兑现 P4 的刷分驱动
- **Player Fantasy**: 不停手连杀把倍率刷满的爽感
- **Input**: 连续击杀（不受伤）
- **Output**: `combo++`；倍率 `= 1 + floor(combo/5)`，上限 8x；受伤清零
- **Success Condition**: 倍率在 HUD 明显跳动，掉连击有视觉警示
- **Failure State**: 倍率增长过慢→无感；过快→分数膨胀失控
- **Edge Cases**: Boss 击杀计入连击但不清零
- **Tuning Levers**: `COMBO_STEP`、`COMBO_CAP`
- **Dependencies**: 得分系统、受伤系统

### 3.7 关卡导演与 Boss
- **Purpose**: 提供中期目标与难度节奏
- **Player Fantasy**: "这关打完了"的阶段性成就感
- **Input**: 时间线驱动的刷怪 + 关底 Boss
- **Output**: 每关先 25s 刷怪阶段，再生成 Boss；Boss 死→过关结算
- **Success Condition**: 关卡时长合理（1–3 分钟），Boss 有可学习的弹幕模式
- **Failure State**: 刷怪空窗期→无聊；Boss 弹幕不可读→劝退
- **Edge Cases**: 第 5/10/15… 关为精英 Boss（血量×1.5、双弹幕）
- **Tuning Levers**: 见下方平衡表 `LEVEL_*` 公式
- **Dependencies**: 全部战斗系统

## 4. 平衡数值表 (Economy Balance)

> 规则：所有数值均为**初始假设**，标记为 `[PLACEHOLDER]` 待实测；每个变量都附"为什么是这个数"的理由，杜绝魔法数字。

```
变量                  | 基准值      | 下限   | 上限   | 理由 / 调优备注
----------------------|-------------|--------|--------|--------------------------------------------------
PLAYER_SPEED          | 260 px/s    | 200    | 320    | 屏幕高 640，约 2.5s 横穿；太快会失去躲弹张力
START_LIVES           | 3           | 2      | 5      | 街机惯例 3 命；命少则劝退，命多则无代价
HIT_INVULN            | 2.0 s       | 1.2    | 3.0    | 给复位缓冲，避免复活即死；过长则受伤无惩罚
LIVES_CAP             | 5           | -      | -      | 1UP 道具上限，防无限堆命
LOOP_DURATION         | 1.3 s       | 0.9    | 1.8    | 招牌闪避窗口；太短躲不掉，太长变外挂
LOOP_COOLDOWN         | 0.4 s       | 0.2    | 0.8    | 后摇，防连滚
LOOP_START/星          | 5 (颗星)    | 1      | 8      | 开局 5 颗星(桶滚充能)，HUD 以 ★ 显示；每关 +1、POW 道具可补；满星时拾取转 +300 分
FIRE_INTERVAL         | 0.18 s      | 0.10   | 0.25   | Tier0 射速；手感测试 0.18s 是否"够密"
WEAPON_TIERS          | 0→1→2→3     | -      | 3      | 1/2/3 路弹道；满级再吃 POW 转 +500 分
POW_DROP_CHANCE       | 0.06        | 0.03   | 0.12   | 普通敌 6% 掉率；太频繁则成长无意义
ENEMY_GRUNT_HP        | 1           | 1      | 3      | 杂兵一击；保证"移动靶"的清脆感
ENEMY_DIVER_HP        | 2           | 1      | 4      | 冲撞型，需两击，强调威胁
ENEMY_SHOOTER_HP      | 3           | 2      | 6      | 远程，需集火，给"优先级"决策
BOSS_HP_BASE          | 60          | -      | -      | 公式 BOSS_HP = 60 + level*25；保证 DPS 匹配
ENEMY_BULLET_SPEED    | 200 px/s    | 140    | 300    | 可读弹速；太快不可躲，太慢无压力
COMBO_STEP            | 每 5 杀 +1x | -      | -      | 倍率增长节奏；太慢无感
COMBO_CAP             | 8x          | 4      | 12     | 防止分数膨胀失控
SCORE_GRUNT           | 100         | -      | -      | 基准分；其余按威胁度缩放
SCORE_BOSS            | 5000+lv*1000| -      | -      | 关底大奖，驱动通关动机
LEVEL_SPAWN_INTERVAL  | 1.2 - lv*0.05 (下限0.35) | - | - | 刷怪密度随关卡收紧
```

**平衡"崩溃"定义（先定失败长什么样）**：
- 若玩家在第 1 关 60 秒内平均阵亡 > 2 次 → 难度过高，下调 `ENEMY_BULLET_SPEED` / `LEVEL_SPAWN_INTERVAL`。
- 若玩家满级武器后单关 Boss < 8 秒被秒 → DPS 溢出，上调 `BOSS_HP_BASE` 系数。
- 若连击倍率长期停在 1x → `COMBO_STEP` 过慢，改为每 3 杀 +1x。
- 若一局 10 分钟分数 > 1,000,000 → 触发膨胀检查，`SCORE_*` 整体 ×0.8。

## 5. 新手引导 (Onboarding)

```markdown
## Onboarding Checklist
- [x] 核心动词（移动+射击）在开局 30 秒内可直接操作
- [x] 第一波为无射击杂兵，保证首杀成功、无失败可能
- [x] POW / Loop / 1UP 各在安全的低密度窗口首次出现并提示
- [x] 玩家至少通过"捡到 POW 火力突变"这一探索性正反馈自学成长
- [x] 首关结束（击败第一个 Boss）即高潮钩子，引导"再来一局/下一关"
```

## 6. 实现备注 (Implementation Notes)

- **零依赖**：全部用 Canvas 2D + Web Audio 合成音效，单文件 `1942.html` 直接双击运行。
- **固定步长**：游戏逻辑用固定 `dt` 累加器，渲染插值，避免不同刷新率下手感漂移。
- **自绘美术**：飞机/敌机/Boss 用程序化几何图形 + 配色绘制，不依赖外部图片资源。
- **操作**：键盘（方向/WASD + J/空格开火 + K/Shift 桶滚）；触屏（拖拽移动 + 自动开火 + 右下角桶滚按钮）。
- **可访问性**：提供音效开关；支持暂停（P / Esc）。
